# M5 scheduling latents — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the three remaining M5 scheduling latents — node-label affinity input path, cross-node `request_id` idempotency with a same-target RPC retry, and a `least-actual-load` placement strategy.

**Architecture:** Three independent, additive features over the existing M5 placement stack (`scheduler` → `coordinator` → node-gated `Provision`). Affinity is resolved at the entry node (no wire change to `ProvisionRequest`). Idempotency adds an in-memory TTL dedup map on the target's `InternalService`. `least-actual-load` gossips actual util on `NodeState` via the existing 10s re-advertise cadence and ranks survivors on it; `fits()` still admits on reserved capacity (no over-admit).

**Tech Stack:** Go 1.25, gRPC + protobuf (`buf` 1.66, remote BSR plugins — needs network), memberlist gossip, testify.

## Global Constraints

- **Go 1.25**; run `gofmt -w` on every changed `.go` file before each commit.
- **Standalone must keep working** — a node with no cluster (no `cluster_secret`/seeds) uses a self-only candidate set; new gossip fields are simply absent.
- **Additive wire evolution only** (ADR-0009): new proto/`NodeState` fields are backward-compatible; old peers omit them and read zero values.
- **Codegen flow:** edit `.proto` → `buf generate` → `go build ./...` → commit regenerated `internal/gen/sbxswarm/v1/*` (git-tracked). `go build` does NOT compile tests — run `go vet ./...` / `go test ./...` to catch test breakage.
- **gopls diagnostics are unreliable after codegen/interface changes** — trust the real `go` toolchain, not the IDE.
- **Secrets invariant:** never log/persist/gossip secret/env values, node keys, or auth tokens. `request_id` (= a generated op id) and util floats are non-sensitive — fine to store/gossip.
- **Authz:** no new gRPC *methods* are added (dedup/affinity/util ride existing RPCs), so the `TestAuthz_AllMethodsClassified` drift-guard is unaffected. Do not add methods.
- **Integration tests use FIXED ports** — if you add one, pick fresh ports (used so far: REST `19443`–`19846`; gossip `17946`–`17999`).
- Branch `m5-latents` off latest `main`; commit per task.

Reference: spec `docs/superpowers/specs/2026-06-17-m5-latents-design.md`, ADR-0012, ADR-0007, ADR-0009.

```go
// dedup is a bounded TTL cache of request_id -> sandbox_id for cross-node
// Provision idempotency: a re-sent request returns the original sandbox.
// ponytail: O(n) oldest-eviction, fine at max<=1024; swap for an LRU if it grows.
type dedup struct {
	mu  sync.Mutex
	m   map[string]dedupEntry
	ttl time.Duration
	max int
}

type dedupEntry struct {
	sandboxID string
	at        time.Time
}

func newDedup(ttl time.Duration, max int) *dedup {
	return &dedup{m: map[string]dedupEntry{}, ttl: ttl, max: max}
}

func (d *dedup) get(id string) (string, bool) {
	d.mu.Lock()
	defer d.mu.Unlock()
	e, ok := d.m[id]
	if !ok || time.Since(e.at) > d.ttl {
		delete(d.m, id)
		return "", false
	}
	return e.sandboxID, true
}

func (d *dedup) put(id, sandboxID string) {
	d.mu.Lock()
	defer d.mu.Unlock()
	now := time.Now()
	for k, e := range d.m { // sweep expired
		if now.Sub(e.at) > d.ttl {
			delete(d.m, k)
		}
	}
	if len(d.m) >= d.max { // evict oldest
		var ok string
		var ot time.Time
		first := true
		for k, e := range d.m {
			if first || e.at.Before(ot) {
				ok, ot, first = k, e.at, false
			}
		}
		delete(d.m, ok)
	}
	d.m[id] = dedupEntry{sandboxID: sandboxID, at: now}
}
```

func (s *InternalService) Provision(ctx context.Context, r *sbxv1.ProvisionRequest) (*sbxv1.ProvisionReply, error) {
	if s.cordoned != nil && s.cordoned() {
		return &sbxv1.ProvisionReply{Accepted: false, Reason: "cordoned"}, nil
	}
	if r.RequestId != "" {
		if sbID, ok := s.dedup.get(r.RequestId); ok {
			return &sbxv1.ProvisionReply{Accepted: true, SandboxId: sbID}, nil
		}

	// Defensive: re-apply the built-in floor at the node->node trust boundary so a
	// peer that sends an unsized spec cannot bypass capacity accounting (ADR-0011).

	rec, err := s.mgr.AdmitAndCreate(ctx, spec)

	if err != nil {
		return nil, err
	}
	if r.RequestId != "" {
		s.dedup.put(r.RequestId, rec.ID)
	}
	return &sbxv1.ProvisionReply{Accepted: true, SandboxId: rec.ID}, nil
}

// callProvisionWithRetry sends a Provision and, on a post-dial RPC error, retries
// the SAME target once — safe because the target dedups by request_id, so a lost
// response will not create a duplicate. It NEVER falls through to another
// candidate after an ambiguous error (a different node does not share the dedup

func callProvisionWithRetry(ctx context.Context, client sbxv1.InternalServiceClient, msg *sbxv1.ProvisionRequest) (*sbxv1.ProvisionReply, error) {
	reply, err := client.Provision(ctx, msg)
	if err != nil && msg.RequestId != "" {
		reply, err = client.Provision(ctx, msg)
	}
	return reply, err
}
